# 数据文件映射管理

  将数据文件和它对应的数据类进行关联起来（就是集中管理，使用 Odin 进行界面制作，数据解析使用的 Json 数据和 Newtonsoft ）

  映射窗口可以通过 Unity 工具栏的 Tools/Custom/DataWindow 打开，效果图如下：

  其中内容绘制是我自己写的一个 Odin 特性实现的（这个特性不允许其他地方用，在特性绘制中使用了固定类（ MapContent ），并不是一个可泛用的特性。），可以删除去使用 Odin 的 TableList 特性，会将 List 中的内容按照表格的形式来进行绘制。效果如下图：

没有快捷打开数据文件功能，可以在 MapContent.OpenContentWindow 方法上添加 Button 特性

  1. 第 1 列的 X 表示删除当前这一行的映射，第一行第一列的 + 表示添加一个映射，和按钮 增加映射关系 效果一样
  2. 第 2 列表示打开当前数据文件，使用对应的数据类（最主要的功能就是这个了，但具体数据类的绘制还是要具体编写，此窗口只是一个集中管理的地方），具体的数据窗口可以看 Window/MapContentWindow 这个类
  3. 第 3-5 列是数据映射文件里面的信息，不可更改（前提是使用了 CustomTableList 特性），第 3 列是数据文件名，第 4 列是文件在 Unity 中的 GUID，第 5 列是对应的映射数据类（看下文）
# 搜索

  搜索是根据映射中的文件名来搜索的，如果需要其他的设置，可以查看 DataFileMapWindow.OnSearchStrValueChange 方法，其中 ShowMap 是用来显示的数据，AllMap 是表示所有数据，需要将 AllMap 中符合搜索条件的存入 ShowMap 中

# 增加映射关系

效果图如下：

  需要一个数据文件和数据映射类型，其中数据映射类型是通过获取程序集 Assembly-CSharp 中的所有继承 IDisplayTreeItem 接口（位于 DataProtoType 文件夹）的类，可以自行更换成其他的接口或者类型（将 IDisplayTreeItem 接口的所有引用都更改就行了），如果想获取其他程序集的内容，可以去 Window\DataFileMapWindow.GetAllDataType 方法进行更改，使用的 Odin 的 ValueDropdown 特性，需要返回一个字符串集合。

# 保存按钮

  一般情况不会派上用场，在增加映射关系和删除映射关系的时候，都会自动保存。

# MapContentWindow

  数据内容窗口，是具体的数据，采用 Json 进行反序列化，需要自己对数据类的样式进行处理，此窗口只是把所有的数据采用 TableList 特性进行绘制，此窗口不会自动保存。

# 其他

  Odin 可以自定义特性以及绘制，可以查看对应的示例，其中要通过名字获取方法的话（例如 OnValueChanged 特性需要的方法名），可以通过 ActionResolver.Get(OdinAttributeDrawer.Property, OdinAttributeDrawer.Attribute.X) 来获取（其中 OdinAttributeDrawer 为你自定义的属性绘制类，X 为你的方法名）

  打开时可能会有错误，一般是因为没有对应的图片，可以通过 Tools/Custom/UnityIconTexture 打开窗口，复制对应的图片名称到报错的地方替换即可。也可以使用 Odin 的 EditorIcons
