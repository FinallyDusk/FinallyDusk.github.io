# GameTimerComponent

  继承了 MonoBehaviour，用于给 GameTimerSystem 更新，所以重要的就只有 Update 里面的内容，其他的都可以随便写，只需要在 Update 函数里面让 GameTimerSystem 更新即可

# GameTimerSystem

  最主要的计时系统，如果要回收继续使用，建议调用一次 InitSystem (一直只使用一次就没有这个必要了)，最主要的方法为 GetTimer ，其他的方法基本不用看，需要的操作都可以从类 GameTimerItem 中找到，需要注意，如果你在计时器执行方法时做了操作（ 添加``暂停``继续``移除 ）, 将会在下一次更新计时器时生效，具体代码如下：

# GameTimerItem

  计时器，不能自己 new 出来，必需要从 GameTimerSystem.GetTimer 方法获取，获取之后需要调用 InitItem 方法，参数有五个：

count 计时器执行次数

intervalTime 执行间隔

callback 每次执行的方法

startTrigger 计时器刚开始调用时是否触发 callback ，默认为 false

controller 计时器控制者，默认为空，不为空时，基础的计时器更新时间会结果 controller.UpdateDeltaTime 方法的控制，以达到单个计时器暂停的效果（和 Pause 方法不同，此方法由 controller 控制，所以可以做到如 war3 中单个单位暂停后，眩晕之类的 buff 持续时间不变的情况，当然也可以调用 Pause 方法，但是这样就表示单位需要持有这个计时器）

方法 SetEndCallback 可以设置计时器结束时的回调（计时器不能强行结束）

外部可以调用的方法有：开始计时器 Start ，暂停 Pause , 继续 Continue
